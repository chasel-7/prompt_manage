# 提案：init-prompt-library-system (初始化提示词词库管理系统)
## 1. Summary
本提案旨在构建一个企业级的**提示词词库管理系统**，服务公司内部 20 人团队。系统包含四大核心模块：提示词管理、前后缀填充、提示词润色、输入分析助手。旨在提升团队使用 AI 工具的效率，规范提示词创作流程，并提供智能化的内容增强能力。

## 2. Why
### Background
- 团队日常使用 AI 频率高，但提示词存储分散（微信、记事本），无法复用。
- 缺乏统一的提示词审核与标准化机制，质量参差不齐。
- 手动重复撰写提示词耗时，且智能化润色与分析能力依赖人工经验，效率低。
### Problem Statement
- 提示词资产无法沉淀复用，导致生产成本高。
- 缺乏可视化工具辅助提示词的拼接、填充与润色。
- 自定义 AI 助手（Agent）能力弱，无法承接复杂业务分析需求。
### Alternatives Considered
- 直接使用第三方 AI 应用（如 ChatGPT 官方）：无法定制内部工作流，数据不安全。
- 自研复杂平台：周期长，维护成本高，对 20 人团队略显笨重。
- **本方案：轻量级前后端分离系统**：快速上线、数据私有化、功能精准匹配。

## 3. What Changes
### 3.1 Scope
**核心目标**：搭建一个支持 CRUD + 收藏的词库平台，集成智能润色与智能分析能力。

#### 模块 A：提示词管理 (Prompt Management)
- **功能**：
  1. **增**：创建提示词分组，上传提示词（含标题、内容、描述）。
  2. **删**：逻辑删除提示词，移入回收站。
  3. **改**：编辑提示词内容与元信息。
  4. **查**：按关键词、标签、分组搜索，支持分页、排序。
  5. **收藏**：标记/取消标记常用提示词，置顶展示。

#### 模块 B：前后缀填充 (Suffix/Prefix Filler)
- **功能**：
  1. 下拉选择已有的提示词模板，作为**前缀 (Prefix)** 或 **后缀 (Suffix)**。
  2. 提供一个文本编辑器，供人工输入补充内容。
  3. **实时预览**：自动拼接前缀 + 人工输入 + 后缀，生成最终完整提示词。
  4. **一键复制**：提供一键复制预览结果的按钮。

#### 模块 C：提示词润色 (Polishing)
- **功能**：
  1. 输入原始提示词文本。
  2. 选择**润色策略**（下拉框）：
     - 策略 1：专业术语增强（补充行业词汇，提升严谨性）。
     - 策略 2：逻辑结构化（分点梳理，提升条理性）。
     - 策略 3：语气具体化（调整指令强度/友好度）。
  3. 点击调用 AI 接口，返回润色后的文本。
  4. 支持对比查看：原文 vs 润色后。

#### 模块 D：输入分析助手 (Input Analysis Assistant)
- **功能**：
  1. **角色配置**：支持自定义系统提示词（System Prompt），定义助手身份（如“资深剧本分析师”、“代码审计官”）。
  2. **交互面板**：类似 ChatGPT 对话窗口。
  3. **任务执行**：用户输入内容，助手根据设定角色完成任务。
     - 典型场景：分析剧本剧情、总结文档、代码逻辑审查、业务需求拆解。
  4. **历史记录**：保存对话历史，支持回溯编辑。

### 3.2 ADDED Requirements
#### Requirement: 系统基础规范
- **技术栈**：后端 Spring Boot / Vue3（前后端分离）。
- **UI 规范**：使用简洁的中后台 UI 组件库（如 Element Plus/Ant Design Vue），保持操作逻辑统一。
- **数据持久化**：MySQL 存储业务数据，支持数据的增删改查。
- **权限控制**：基于角色的权限管理（RBAC），限制不同角色对模块的访问（例如：普通用户只能使用，管理员维护词库）。

#### Requirement: 模块 A - 提示词管理细节
- 每个提示词必须包含：标题（必填）、内容（必填）、描述（选填）、标签（多选）、所属分组。
- 搜索功能支持对标题、描述、内容内容的模糊检索。
- 收藏功能优先级最高，收藏列表在首页优先展示。

#### Requirement: 模块 B - 前后缀填充细节
- 填充组件支持“清空”、“交换前后缀”快捷操作。
- 输入框支持快捷键（如 Ctrl+Enter）触发预览刷新。
- 预览区必须支持代码高亮（如果提示词包含代码）。

#### Requirement: 模块 C - 提示词润色细节
- 提供“历史润色记录”功能，记录每次润色的策略与结果。
- 支持将润色结果**一键保存**到“提示词管理”词库中。
- AI 调用接口需包含重试机制，防止网络抖动导致失败。

#### Requirement: 模块 D - 输入分析助手细节
- 支持上传文件（TXT/MD/PDF）作为输入上下文。
- 支持导出对话结果为 Markdown 文档。
- 系统提示词编辑器支持语法高亮与模板变量（如 `{{user_input}}`）。

## 4. Technical Constraints
- **API 设计**：遵循 RESTful 风格，接口返回标准 JSON 格式（code, msg, data）。
- **AI 接口**：预留通用 AI 调用接口，适配主流大模型（如 OpenAI、Anthropic、国内大模型），支持配置密钥。
- **状态管理**：前端使用 Pinia 管理全局状态，模块间状态隔离。
- **安全性**：用户密码加密存储，API 调用权限校验，防止越权访问。

## 5. Non-Goals
- 暂不支持多语言实时翻译功能（核心聚焦中文提示词管理）。
- 暂不支持复杂的工作流审批（仅管理员即可维护词库）。
- 暂不开发移动端适配（优先 PC 端高效使用）。
